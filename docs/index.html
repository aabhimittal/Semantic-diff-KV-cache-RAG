semantic-kv

A KV cache for RAG keyed on semantic similarity of retrieved chunks — reuse the shared span, compute only the delta.

GitHub Benchmark HF Space demo

The problem

Every turn of a RAG conversation re-encodes near-identical context. Retrieval returns 80%-overlapping chunks, and the model pays full prefill for all of them, again. Exact-match prefix caches (vLLM, SGLang radix trees) go cold the moment one chunk is reordered or lightly edited — and retrieval reorders and lightly edits constantly.

turn 1:  [chunk A][chunk B][chunk C][query₁]   full prefill, segments cached
turn 2:  [chunk A][chunk B'][chunk D][query₂]  A reused · B' served from B (cos 0.94)
                                               only D + query computed

The caveat that is the whole project

The KV of a shared span is not reusable for free when the preceding tokens differ:

  1. Position. RoPE rotates keys by absolute position — but rotations compose, so a cached key moves from position p to q exactly, with one extra rotation R(q−p). Solved, not approximate.
  2. Cross-attention. A cached chunk was encoded standalone; its hidden states never attended to the chunks now in front of it. This is a real approximation that never fully goes away.
The project therefore ships with a divergence harness that measures exactly when the approximation holds, and a repair knob: recompute_ratio re-runs the leading fraction of each reused chunk against its true prefix. At 1.0 the pipeline is numerically identical to a vanilla forward pass — asserted by the test suite — so the approximation lives entirely in one tunable knob.

Architecture

A vector store fronts the KV cache — and it indexes cache fragments, not documents. One point per cached chunk segment; nearest-fragment search implements the semantic key; payload filters (doc_id) implement span-level invalidation. In-memory store by default, Qdrant adapter included.

lookup(chunk):  exact (text hash) → semantic (cos ≥ τ) → miss
assemble():     reuse  → RoPE re-rotation to new position
                blend  → leading r·L tokens recomputed vs true prefix
                miss   → fresh compute + standalone encode for next time
invalidate(doc_id=…) / invalidate(chunk_id=…)   # span-level
